# منصة إدارة المتابعة والاعتماد

## الهدف
بناء منصة مؤسسية موحدة وقابلة للتوسع لإدارة أعمال المتابعة والاعتماد في المجلس الصحي السعودي.

## الرؤية
- المنصة متعددة مساحات العمل.
- كل عمل Workspace مستقل.
- كل Workspace يحتفظ بمنطق العمل والحقول ودورة الاعتماد الخاصة به.
- الوظائف المشتركة تبنى مرة واحدة في نواة المنصة.
- لا يجوز اختزال المشروع في الرقابة المالية والركائز فقط.

## التقنية
- React
- TypeScript
- Vite
- Supabase
- GitHub Pages
- GitHub Actions
- Responsive للجوال والكمبيوتر
- Modular Monolith
- RLS
- RBAC
- Audit Log

## القواعد الحاكمة
- جميع الأعمال السابقة وملفات HTML وExcel وPDF وDOCX أصول للمشروع.
- لا يجوز تجاهل أو حذف أو استبدال أصل سابق دون تحليله وتوثيق البديل واعتماده.
- البيانات الأصلية والتأسيسية ليست بيانات وهمية.
- لا تستخدم بيانات تجريبية مكان البيانات الفعلية.
- Supabase هو المصدر المؤسسي للبيانات.
- لا تعديل لقاعدة البيانات قبل اعتماد Architecture وERD.
- لا حذف لوظيفة قديمة قبل نقل بياناتها واختبار البديل.
- لا commit أو push دون موافقة صريحة من مالك المشروع.
- يجب تشغيل npm run build بعد كل تعديل برمجي.
- يجب اختبار الكمبيوتر والجوال في كل مرحلة.
- كل مهمة تكون صغيرة ومحددة وقابلة للمراجعة والتراجع.
- يجب التوقف بعد كل مرحلة للمراجعة.

## مساحات العمل
1. تقرير الكفاءة الرقابية
2. ركائز كفاءة الإنفاق
3. ملاحظات الديوان العام للمحاسبة
4. الأهداف التشغيلية
5. سجل المخاطر
6. أوامر الصرف
7. أعمال اللجان
8. المشاريع التحسينية

## الرقابة المالية
- المرجع الوظيفي هو ملف منصة متابعة الكفاءة الرقابية السابق.
- المرجع الرسمي هو تقرير تقييم مستوى الكفاءة الرقابية.
- بيانات الرقابة المالية بيانات تأسيسية معتمدة.
- يجب الحفاظ على 33 ملاحظة رقابية ودورة الموظف والمدير والاعتماد والإرجاع والأدلة والمراسلات وسجل الحركات والمؤشرات.

## ركائز كفاءة الإنفاق
- المرجع هو دليل ركائز كفاءة الإنفاق وبيانات Supabase.
- الهيكل: 7 ركائز، 22 ركيزة فرعية، 33 سؤالًا، 73 معيارًا.
- دورة العمل: تقييم ذاتي، اعتماد مدير المركز، مراجعة المختص، الإرجاع والتعديل، اعتماد المختص، اعتماد المشرف العام.

## App.tsx
- لا يحتوي على جميع منطق المنصة.
- لا يحتوي على بيانات تشغيلية ثابتة.
- يكون مسؤولًا عن التهيئة والتوجيه العام فقط.
- كل مساحة عمل تفصل داخل feature مستقلة.

## التعامل مع الأدوات
- لا تحذف ملفات.
- لا تنشئ بيانات وهمية.
- لا تنشئ جداول من تلقاء نفسك.
- لا تعدل Supabase دون موافقة.
- لا تعمل commit أو push.
- اعرض الملفات التي ستعدلها قبل التنفيذ.
- اعرض نتيجة البناء والاختبار بعد التنفيذ.

## قاعدة التعارض
عند التعارض بين أي اقتراح وهذه التعليمات، تقدم هذه التعليمات، ولا يتم التغيير إلا بموافقة صريحة من مالك المشروع.
